# ScoreTuner 설계 문서

날짜: 2026-07-19
상태: 승인됨

## 개요

기타 연습을 위한 안드로이드용 웹앱(PWA). 두 가지 핵심 기능을 제공한다:

1. **튜너** — 마이크로 기타 줄 소리를 분석해 정확한 조율을 안내
2. **악보** — 노래를 검색·입력하면 기타로 치기 쉬운 코드로 편곡해 라이브러리에 보관하고, 연주를 인식해 악보가 따라 넘어가는 기능

## 대상 환경

- 갤럭시 S23 이상 안드로이드 폰, 태블릿 (Chrome 브라우저)
- 반응형: 폰 세로/가로, 태블릿 큰 화면 모두 지원
- PWA: 홈 화면 설치, 오프라인 실행(악보 열람·튜너는 오프라인 동작)

## 기술 구조

- **순수 웹 기술**: HTML + CSS + JavaScript(ES 모듈), 빌드 도구 없음
- **배포**: GitHub Pages (HTTPS — 마이크 권한에 필수)
- **저장소**: IndexedDB(악보 라이브러리), localStorage(설정). JSON 내보내기/가져오기로 백업
- **오디오**: Web Audio API — 피치 검출(튜너), 크로마그램 분석(연주 인식), 칼림바 음 합성
- **PWA**: manifest.json + Service Worker(오프라인 캐싱)
- **폰트**: Pretendard (CDN + 오프라인 캐싱)

### 화면 구성 (5개)

| 화면 | 역할 |
|------|------|
| 튜너 | 줄 선택, 기준음 재생, 실시간 조율 안내 |
| 라이브러리 | 저장된 악보 목록 (가나다순/알파벳순), 검색 |
| 악보 뷰어 | 가사+코드 표시, TAB 모드, 키 조절, 연주 모드 진입 |
| 곡 추가 | 붙여넣기 모드 / AI 모드로 새 악보 생성 |
| 설정 | API 키, 기준 주파수, 테마 |

### 디자인

- 모던·미니멀, 전문 앱 수준의 완성도
- 다크/라이트 테마 — 시스템 설정 따라 자동 전환 (수동 강제 선택도 설정에서 가능)
- 다크: 짙은 배경(#0f1115 계열) + 그린 포인트, 라이트: 밝은 배경 + 블루 포인트
- UI 언어: 한국어

## 기능 1: 튜너

### 기준음 재생
- 표준 튜닝: 6번줄 E2(82.41Hz) ~ 1번줄 E4(329.63Hz), 기준 A4=440Hz(설정 변경 가능)
- 기타 헤드스톡 이미지 + 6개 줄 버튼 UI
- 줄 버튼 터치 시 **칼림바 음색**으로 해당 음 재생 — Web Audio 합성(기음 + 배음 2~3개, 빠른 어택/긴 감쇠 엔벨로프)

### 실시간 조율
- 마이크 입력 → 자기상관(autocorrelation, YIN 계열) 피치 검출
- 튕긴 음에 가장 가까운 줄 **자동 감지** (수동 선택 시 해당 줄 고정)
- 표시: 현재 음이름(예: A2), 센트 단위 편차 게이지(-50 ~ +50센트)
- 안내: 낮으면 "조여서 높이세요 ▲", 높으면 "풀어서 낮추세요 ▼"
- **±5센트 이내**: 초록색 "정확합니다" 표시 + 진동(Vibration API) 피드백

## 기능 2: 악보

### 곡 추가 — 하이브리드 방식

**붙여넣기 모드 (기본, 무료)**
1. 노래명·가수명 입력
2. 앱이 검색 링크(구글 등) 버튼 제공 → 사용자가 코드+가사 텍스트 복사
3. 붙여넣으면 파서가 코드 줄/가사 줄을 자동 구분해 구조화

**AI 모드 (API 키 등록 시)**
- 설정에서 Claude API 키 등록 (`anthropic-dangerous-direct-browser-access` 헤더로 브라우저에서 직접 호출)
- 노래명·가수명만 입력하면 코드 진행 + 가사 + 섹션 구조 + 반주 패턴을 AI가 생성
- AI가 생성한 가사는 부정확할 수 있음을 안내, 수동 편집 가능

### 편곡 엔진 (순수 함수, 두 모드 공통)

- **허용 코드 집합**: A, Am, A7, Bb, Bm, Bm7, C, C7, D, Dm, D7, Dm7, E, Em, E7, F, Fm, G, G7, Gm
- 원곡 코드 진행 분석 → 12개 이조 후보 중 허용 집합 포함률이 최대인 키 선택
- 이조로 해결 안 되는 코드는 근접 단순화 (예: C#m7 → 이조 후 Em, sus4/add9 → 기본형)
- 카포 사용이 유리하면 카포 위치 제안 (예: "카포 2프렛, C 포지션")

### TAB 생성 (곡 저장 시 함께 생성)

- **코드 다이어그램**: 내장 운지 DB(허용 코드 20개 + 예외 대비 확장)에서 SVG로 렌더링
- **반주 패턴**: 내장 패턴 라이브러리(스트로크 4종 + 아르페지오 4종, 4/4·3/4·6/8)에서 곡 분위기·박자에 맞는 패턴 선택(AI 모드는 AI가 지정, 붙여넣기 모드는 사용자가 선택, 기본값 제공) → 6줄 TAB으로 렌더링

### 악보 뷰어

- **기본 모드**: 가사 위에 코드가 정렬된 형태 (섹션 구분: Intro/Verse/Chorus 등)
- **TAB 모드 토글**: 상단에 사용 코드 다이어그램 모음 + 섹션별 반주 패턴 TAB 추가 표시
- **키 조절**: ▲/▼ 반음 단위 (노래방식). 조절 후 편곡 엔진 재적용으로 쉬운 코드 유지, 어려워지면 카포 제안. 곡별 키 설정은 저장됨
- 폰트 크기 조절

### 라이브러리

- 정렬: 한글 제목 가나다순 → 영문 알파벳순 (localeCompare)
- 검색(제목/가수), 삭제, 수동 편집
- JSON 파일 내보내기/가져오기 (백업)

## 기능 3: 연주 모드

- 진입 시 **가로 화면**(Screen Orientation API + 미지원 시 CSS 회전 폴백), **Wake Lock**으로 화면 꺼짐 방지
- **연주 인식 자동 진행**:
  - 곡의 코드 진행 시퀀스를 미리 알고 있음 → "다음 예상 코드가 연주됐는가"만 판별하는 시퀀스 기반 매칭
  - 마이크 입력 → FFT → 12음 크로마 벡터 → 예상 코드 템플릿과 코사인 유사도 매칭
  - 오탐 방지: 연속 N프레임(약 0.3~0.5초) 일치 시에만 전환, 앞으로만 이동(역주행 방지)
  - 현재 위치 하이라이트 + 자동 스크롤(현재 코드가 화면 중앙 유지)
- **수동 보정**: 화면 좌측 터치 = 한 코드 뒤로, 우측 터치 = 한 코드 앞으로
- 노래+연주 동시 상황에서 인식률이 떨어질 수 있으므로 수동 보정을 항상 활성 상태로 유지

## 데이터 모델

```
Song {
  id, title, artist,
  originalKey, currentKey, capo,
  timeSignature, pattern,        // 반주 패턴 id
  sections: [{
    name,                        // Verse, Chorus...
    lines: [{ chords: [{chord, position}], lyrics }]
  }],
  createdAt, updatedAt, source   // 'paste' | 'ai'
}
```

## 오류 처리

| 상황 | 처리 |
|------|------|
| 마이크 권한 거부 | 안내 화면 + 권한 재요청 버튼 (튜너·연주 인식 외 기능은 정상 동작) |
| API 키 미등록/오류/한도 초과 | 안내 메시지 + 붙여넣기 모드 유도 |
| 붙여넣기 파싱 실패 | 인식된 부분까지 보여주고 수동 편집 화면으로 폴백 |
| 저장 실패(용량 등) | 오류 안내 + 내보내기 백업 권장 |
| 오프라인 상태 | 튜너·라이브러리·뷰어·연주 모드 정상 동작, 곡 추가(AI/검색)만 온라인 필요 안내 |

## 테스트 전략

- **단위 테스트** (Node 기반, 순수 함수): 편곡 엔진(이조·단순화·카포), 붙여넣기 파서, 피치→음이름 변환, 정렬 로직
- **오디오 테스트**: 합성 사인파/코드 샘플로 피치 검출·크로마 매칭 정확도 검증
- **실기기 검증**: S23에서 마이크 튜닝, 연주 인식, 가로 모드, PWA 설치, 오프라인 동작 확인

## 범위 제외 (YAGNI)

- 계정/클라우드 동기화 (JSON 백업으로 대체)
- 멜로디 TAB 자동 채보
- 메트로놈, 녹음 기능
- 다국어 UI
